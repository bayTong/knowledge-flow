# 知识库规范

> 本文档定义了 [知识库名称] 的结构规则、命名约定、标签体系和质量标准。
> 所有 Agent 在操作本知识库前，必须先阅读本文档。
> 最后修订：YYYY-MM-DD

---

## 一、领域范围

本知识库覆盖以下领域：

1. **[领域 A]**——[一句话说明]
2. **[领域 B]**——[一句话说明]（如只有一个领域，删除本行）

---

## 二、目录结构

```
<知识库根目录>/
├── raw/
│   ├── articles/          ← 第一层：知识原材料（只读不可改）
│   ├── papers/            ← 论文原文（只读）
│   ├── transcripts/       ← 对话/访谈记录（只读）
│   ├── assets/            ← 图片、图表等附件
│   └── _reading-maps/     ← 粗读器产出的阅读地图
├── wiki/                   ← 第二层：Agent 整理后的结构化知识
│   ├── entities/          ← 实体页
│   └── queries/           ← 查询存档
├── schema/                 ← 第三层：治理整个知识库
│   ├── SCHEMA.md          ← 本文件（规范）
│   ├── index.md           ← 导航中枢（按模块索引所有页面）
│   └── log.md             ← 操作日志（仅追加）
└── _archive/              ← 归档页面（过时但保留）
```

### 模块编号规则

- 两位数字 + 连字符 + 中文模块名，如 `01-核心架构`
- 新增模块时按顺序递增编号
- 模块目录下直接存放页面文件，不设二级子目录

---

## 三、文件命名

### 页面文件

- **使用中文文件名**，以页面标题命名
- 含空格不含连字符，如 `记忆系统四层架构.md`
- 文件名必须与 frontmatter 中 `title` 的主标题部分一致

### 管理文件

- `SCHEMA.md`、`index.md`、`log.md` 使用英文大写名，三者不出现在关系图谱中

### raw/ 原始资料文件

- `articles/`：`作者-主题-年份.md`（如 `karpathy-llm-wiki-2025.md`）
- `papers/`：`作者-年份-简短标题.md`
- `transcripts/`：`YYYY-MM-DD-主题.md`
- raw/ 下所有文件只读不可修改。纠错或更新写入 wiki 页面，不动 raw/

### 禁止

- 英文小写连字符命名
- 文件名与 frontmatter `title` 不一致

## 四、Frontmatter 规范

每个页面文件必须以 YAML frontmatter 开头：

```yaml
---
title: [模块名 · 页面主标题]
created: YYYY-MM-DD
updated: YYYY-MM-DD
type: concept | comparison | entity | query
tags: [来自下方标签体系]
sources: [信息来源路径或描述]
confidence: high | medium | low
contested: false                 # 可选。true 表示页面存在未解决的矛盾信息
---
```

> **title 格式按页面类型区分**：
> - `concept` / `comparison`：`模块名 · 页面主标题`
> - `entity`：`实体 · 实体名称`
> - `query`：`问答 · 问题简述`

### 字段说明

| 字段 | 必填 | 说明 |
|------|:---:|------|
| `title` | ☑ | 格式见上方按类型区分规则，主标题部分与文件名一致 |
| `created` | ☑ | 页面首次创建日期 |
| `updated` | ☑ | 最后更新日期，每次修改内容时必须同步更新 |
| `type` | ☑ | `concept`（机制/原理/架构）、`comparison`（对比分析）、`entity`（工具/模型/人物/组织）、`query`（深度问答存档） |
| `tags` | ☑ | 来自下方标签体系的标签列表，不在体系中的标签不得使用 |
| `sources` | 推荐 | 记录信息来源，可为源码路径、URL、或对话描述 |
| `confidence` | 推荐 | `high`（多来源验证或有源码支撑）、`medium`（单来源或二手信息）、`low`（推测或未验证） |
| `contested` | 可选 | `true` 表示页面存在未解决的矛盾信息，无矛盾时省略 |

### 字段设计原则

- `title` 比文件名多出模块前缀，用于 Dataview 查询和内联属性展示
- `type` 不进入 `tags` 列表，避免冗余
- `confidence` 为 `low` 的页面会在 Lint 时优先标记供审阅
- `contested` 为 `true` 的页面会在 Lint 时优先标记供审阅

## 五、标签体系

新增标签时必须先在此注册，后使用。防止标签无节制蔓延。

### 主题标签（页面在讲什么）

| 标签 | 含义 |
|------|------|
| [标签1] | [含义说明] |
| [标签2] | [含义说明] |
| [至少填 3 个与领域直接相关的标签] | |

### 性质标签（页面有什么特殊属性）

| 标签 | 含义 |
|------|------|
| `comparison` | 页面包含对比分析（多维表格） |
| `decision` | 页面给出了明确的选择建议或最佳实践 |

> 初始标签只定义当前原料会用到的。原料增加后按需扩充，避免提前设计用不上的标签。

## 六、Wikilink 规范

### 格式

```
[[文件名|模块名 · 页面标题]]
```

- 管道符 `|` 前为实际文件名（不含路径、不含 `.md` 扩展名）
- 管道符 `|` 后为完整 frontmatter `title`，用于链接显示文字
- 管道符及显示文字为**必填**，不允许使用无管道的简写形式

### 交叉引用要求

- `concept` / `comparison` / `query` 页面至少链接 2 个其他页面
- `entity` 页面不强制出站链接数量，但必须有 ≥1 个 `concept` 页面引用它（入站链接），否则不创建
- 「相关页面」段落统一放在文件末尾
- 不自链接（页面不在自己的「相关页面」中引用自己）

## 七、导航与日志

### index.md

- `concept` / `comparison` 页面按模块分组；`entity` 页面归入单独的「实体」段；`query` 页面归入单独的「问答」段
- 每个页面一行：`[[文件名|title]] — 一句话摘要`
- 更新任一页面后需同步更新 index 中的摘要（如摘要不再准确）
- 新增/删除页面必须更新 index

### log.md

- 仅追加，不修改历史条目
- 格式：`## [YYYY-MM-DD HH:MM:SS] 操作类型 | 主题`
- 操作类型：`create`、`update`、`restructure`、`fix`、`rename`、`archive`、`lint`
- 超过 500 条时轮转：重命名为 `log-YYYY.md`，新建空白 `log.md`

---

## 八、页面质量规则

### 创建新页面

- 该主题在当前知识库中尚无页面覆盖
- 或已覆盖但新内容足以构成独立子主题（如拆分某个子机制）

### 追加到已有页面

- 新信息是对已有内容的补充或纠错，不足以独立成页

### 不创建

- 顺带提及、次要细节、领域外内容

### 拆分

- `concept` / `comparison` 页面超过约 300 行时考虑拆分为子主题
- `entity` 页面不拆分（实体卡片保持完整）；`query` 页面通常不拆分（问答保持上下文完整性）

### 归档

- 内容完全被新版取代 → 移入 `_archive/` → 从 index 移除 → 更新所有指向它的 wikilink

---

## 九、更新与冲突处理

内容过时或出现矛盾时：

1. 检查新旧信息来源的日期——新来源通常取代旧来源
2. 若双方都有依据且无法判断，记录双方观点（附日期和来源）
3. 在更新后的内容中注明变更原因
4. 如矛盾未解决，在 Lint 报告中标记

---

## 十、图谱显示

- `index.md`、`SCHEMA.md`、`log.md` 通过 `graph.json` 过滤，不显示在关系图谱中
- `raw/` 下所有文件不显示（只读原料，无 wikilink，孤立节点无意义）
- 图谱节点显示中文文件名
- 过滤配置：`"search": "-path:schema -path:raw"`
