# 外置第二大脑 · 战略愿景

> 本文档描述外置第二大脑项目的核心理念和架构总览。
> 详细执行计划见 [`docs/build-plan.md`](build-plan.md)。

---

## 什么是外置第二大脑

不是「AI 帮你记笔记」，而是 **AI 智能体作为持续运行的认知外挂**：

- **摄入管线**：AI 穷举提取 → 独立覆盖审计 → 人审 → 策展入库。覆盖审查与提取分离——飞行员和塔台是不同的人。
- **持久记忆**：知识库作为 AI 的跨会话长期记忆。AI 在对话中主动检索已有知识，而非从零开始。
- **元认知监控**：独立 Agent 定期扫描知识库健康度——遗漏、漂移、盲区、结构趋势。
- **自我进化**：系统从人的审核反馈中学习自己的盲区，持续改进提取和策展质量。

---

## 核心原则

| 原则 | 含义 |
|------|------|
| **AI 做覆盖，人做判断** | AI 穷举提取不遗漏；人决定什么重要、什么该入库 |
| **写入和检查分离** | 摄入 Agent 做提取+写入（摄入会话）；独立 Cron Agent 做语义检查和趋势分析（独立会话）。避免上下文污染 |
| **成本分层** | 最终产出用最好的模型；辅助检测用便宜模型；机械任务不用 LLM |
| **只报告，不自动修** | MC-001 发现的问题只标记和报告，不自动修改内容。所有「该不该做」交给人决策 |
| **SCHEMA 事后涌现** | SCHEMA 不前置定义——从通用模板起步，随每次摄入从原料中自动建议新标签、新模块，经人确认后注册 |

## LLM 的四项根本局限与系统对策

这四个局限是当前 LLM 技术的硬边界，架构不能「解决」它们，只能**降低伤害**、**把不可消除的部分交还给人类**。

| 局限 | 含义 | 架构对策 |
|------|------|---------|
| 无真正语义理解 | LLM 做统计模式匹配，不是推理 | 多轮次提取（分治降低认知负担）+ 提取接口（强迫系统化扫描）+ 聚类辅助（发现 LLM 未标注的关联）+ MC-001 独立交叉验证 |
| 长期连贯思考丢失 | 跨数周复杂推理链丢失上下文 | 知识库作为外部持久记忆 + MC-001 趋势视图 + 概念漂移追踪 + 主动检索注入上下文 |
| 无自主价值判断 | LLM 无法判断「值不值得深挖」 | 人审设在关键决策点 + MC-001 只报告不决策 + 聚类建议只做提示 |
| 无真正创造性连接 | LLM 只在表层做相似度匹配 | MC-001 跨页面/跨 KB 关联发现（AI 做大规模比对）+ 人做创造性判断 + 联合索引降低发现门槛 |
