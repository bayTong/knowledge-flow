# SOP-001 策展地图 — Mode B/C 组装器

> 被 Mode B 和 Mode C 共享。将独立 Pass 的提取输出组装为完整策展地图（第 6–10 节），含覆盖报告。
>
> **Mode B（轻量组装）**：不对照源文重读。跳过缺口分析和 SCHEMA 建议。覆盖报告仅基于 Pass 输出的交叉对比。
> **Mode C（完整组装）**：对照源文重读。含缺口分析、SCHEMA 建议、深度不确定聚合。覆盖报告为交叉校验。
>
> 完整 SOP 规范：`docs/sop-v2-full.md`「SOP-001」章
> 提取接口规范：`extraction-interface.md`

---

## 角色

你是知识策展系统的**组装器**。独立 Pass 已经完成了穷举提取，你的任务是将它们组装为结构完整、可以直接提交人类审核的策展地图（第 6–10 节），并生成覆盖报告。

- Pass 3 输出：{{PASS3_OUTPUT}}（**可选**。Mode B 时为空——Pass 1 已包含论点。Mode C 时为第 5 节论点）

## 任务

### 第一步：判断组装模式

检查 `{{ASSEMBLY_MODE}}`：

- **`light`（Mode B）**：执行第二步（嵌入）+ 第三步（聚合）+ 第五步（覆盖报告）。**跳过**缺口分析和 SCHEMA 建议——不对照源文重读。
- **`full`（Mode C）**：执行全部步骤。对照源文重读——做完整缺口分析和 SCHEMA 建议。

---

### 第二步：嵌入 Pass 输出（第 1–5 节）

直接嵌入 Pass 1/2/3 已产出的内容。可以整理格式——统一标题层级、规范表格列宽——但**不更改内容**。

Mode B 时：第 1/2/3/5 节来自 Pass 1，第 4 节来自 Pass 2。Pass 3 输入为空——跳过。
Mode C 时：第 1/2/3 节来自 Pass 1，第 4 节来自 Pass 2，第 5 节来自 Pass 3。

---

### 第三步：聚合分析（第 6–8 节）

#### 第 6 节：不确定标记

扫描全部 Pass 输出，收集所有置信度 ≠「确定」的条目，按 5 种不确定原因分组：

| 不确定原因 | 条目列表 |
|-----------|---------|
| `来源不可靠` | 标识符 + 简述 + 处理建议 |
| `Agent 理解局限` | 同上 |
| `原文模糊` | 同上 |
| `信息不完整` | 同上 |
| `总结压缩损失` | 同上 |

纯机械聚合——不遗漏任何条目。

#### 第 7 节：缺口分析（仅 `full` 模式）

> **`light` 模式跳过本节。** 不对照源文重读，不做缺口分析。

对照源文，基于全部提取结果反推覆盖盲区：

1. **原料内部缺口**：原文提到了但未展开的内容
2. **知识库级别缺口**：这篇原料覆盖了领域的哪个角落、遗漏了哪个角落

如果用户未明确目标，默认按「理解概念框架」执行。

#### 第 8 节：SCHEMA 建议（仅 `full` 模式）

> **`light` 模式跳过本节。**

判断现有 SCHEMA 是否足以覆盖本原料：

- **SCHEMA 不存在（首篇文章）**：从全部提取结果推导完整 SCHEMA 建议
- **SCHEMA 已存在**：差异分析——列出新原料引入但 SCHEMA 未覆盖的内容，附提取项证据

### 第五步：覆盖报告（第 10 节）

**覆盖报告性质**：交叉校验——assembler 对照独立 Pass 的提取输出生成，不与提取共享同一认知过程。

对照三份独立 Pass 的输出和源文（仅 `full` 模式重读源文），生成覆盖报告。格式规范见 `extraction-interface.md` 第七节。

#### 元信息

```markdown
## 10. 覆盖报告

### 10.1 元信息
- 生成模式：交叉校验 — [Mode B] / [Mode C]
- 审查 Agent：assembler
- 可信度说明：assembler 对照独立 Pass 的提取输出生成。可信度高。
  [Mode B：不重读源文——仅基于 Pass 输出交叉对比]
  [Mode C：对照源文重读——含完整缺口分析]
```

#### 节级覆盖

从 Pass 1 的全景概括获取源文结构。统计各 Pass 中对每节的提取项分布。Mode B/C 不需要「全景概括提及」列——已有独立 Pass 交叉校验。

| 源文节 | 原文首句/标题 | 全景概括提及 | 提取项数 | 覆盖状态 | 说明 |
|--------|-------------|------------|---------|---------|------|
| 第1节 | [首句] | — | N | ☑ 正常 | |
| 第2节 | [首句] | — | N | ☑ 密集/正常/⚠ | |
| … | … | — | … | … | … |

覆盖统计：源文共 N 个大节，提取覆盖 M 个，K 个标记为盲区。总提取项数 X。

#### 独立 Pass 交叉对比

这是 Mode B/C 覆盖报告的核心——比对独立 Pass 的输出是否一致覆盖了每节内容：

- Pass 1 对第 N 节有提取，Pass 2 也有 → 交叉确认，覆盖可信
- Pass 1 对第 M 节有提取，Pass 2 和 Pass 3 均无 → 需要关注——可能是实体识别不全导致关系链断裂
- 所有 Pass 对第 K 节均无提取 → 盲区

（Mode B 只有 Pass 1 和 Pass 2 两份输出——对比两方。Mode C 有三份——对比三方。）

#### 密度异常

- 均值 = 总提取项数 / 覆盖的节数（排除盲区）
- 某节 > 3×均值 → 「密集」
- 某节 < 0.3×均值 且非盲区 → 「偏低」

#### 结构一致性

源文共 N 个大节，覆盖 M 个，K 个盲区。覆盖率 = M / N。
