## 任务

### 第 1–5 节：嵌入三份 Pass 输出

直接嵌入 Pass 1/2/3 已产出的第 1–5 节内容。可以整理格式——统一标题层级、规范表格列宽——但**不更改内容**。

### 第 6 节：不确定标记

扫描 Pass 1、Pass 2、Pass 3 全部输出，收集所有置信度 ≠「确定」的条目。按 5 种不确定原因分组汇总：

| 不确定原因 | 条目列表 |
|-----------|---------|
| `来源不可靠` | 列出所有属于此类别的条目（含标识符 + 简述 + 处理建议） |
| `Agent 理解局限` | 同上 |
| `原文模糊` | 同上 |
| `信息不完整` | 同上 |
| `总结压缩损失` | 同上 |

> 这是纯机械聚合——不需要语义判断。不遗漏任何一条不确定条目。

### 第 7 节：缺口分析

基于全部提取结果（第 3/4/5 节），反推原料的覆盖盲区。你已经知道了原料中「提取了什么」，才能判断「缺了什么」

分两个子维度：

1. **原料内部缺口**：原文提到了但未展开的内容。例如：承诺给出代码但只写了概念、提到某技术但一笔带过、给出结论但未给推导过程
2. **知识库级别缺口**：以系统性理解为目标，这篇原料覆盖了领域的哪个角落、遗漏了哪个角落。例如：一篇讲 RAG 切块策略的文章，覆盖了切块但未涉及嵌入模型选型和评估指标体系

如果用户未明确目标，默认按「理解概念框架」执行，并在分析开头标注

### 第 8 节：SCHEMA 建议

判断现有 SCHEMA 是否足以覆盖本原料的全部内容：

- **SCHEMA 不存在（首篇文章）**：从全部提取结果中推导完整 SCHEMA 建议——页面类型（名称 + 对应目录）、关系类型（边类型 + 含义）、标签分类法（主题标签 + 性质标签）
- **SCHEMA 已存在**：做差异分析——列出新原料引入但 SCHEMA 未覆盖的类型/关系/标签，给出具体建议。每个建议必须有对应的提取项作为证据

### 第 9 节：Agent 签名

**提取统计**：统计 Pass 1/2/3 的全部产出项数量（实体按层分、关系分显式/隐式、论点分类型）

**覆盖声明**：对于超长原料（> 20000 字），列出未逐段细读的区域及原因；常规原料声明已全覆盖
