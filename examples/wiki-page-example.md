# Agent（智能体）

## 是什么

Agent 是一种让大语言模型具备**自主感知、规划、执行和反思能力**的系统架构。与传统的"一问一答"模式不同，Agent 能够将复杂的用户目标分解为多步骤任务，主动调用外部工具（如搜索引擎、计算器、API），并根据工具返回的结果动态调整后续行动策略。

它解决的核心问题是：大模型的能力边界局限——模型本身只能"说"不能"做"（无法直接操作外部系统、处理复杂多步骤任务）。

单智能体公式：**大模型 + 系统提示词 + 工具调用 + 自主工作流（规划→执行→反思）**。

## 核心机制

### 工作流：感知→规划→执行→反思

```
用户目标 → 意图感知 → 任务规划 → 工具选择与调用 → 观察结果 → 反思调整 → 循环直到完成
```

### 关键能力

| 能力 | 说明 |
|------|------|
| **规划（Planning）** | 将复杂目标分解为子任务序列，确定先后顺序和依赖关系 |
| **工具调用（Tool Use）** | 基于当前状态选择最合适的工具（API/数据库/搜索引擎），生成结构化调用参数 |
| **记忆（Memory）** | 维护短期（对话上下文）和长期（跨会话知识）记忆 |
| **反思（Reflection）** | 评估行动结果是否达成目标，失败时调整策略重试 |

### Agent 执行范式

| 范式 | 核心 | 最佳场景 |
|------|------|---------|
| **ReAct** | 思考→行动→观察 循环 | 工具使用、交互式任务 |
| **CoT（思维链）** | 逐步展示推理过程 | 数学、逻辑问题 |
| **ToT（思维树）** | 多路径探索+评估回溯 | 创意、策略问题 |
| **Plan-and-Execute** | 先规划后执行 | 复杂多步骤任务 |
| **Reflexion** | 从错误中学习，存入长期记忆 | 需要试错的任务 |
| **GoT（图推理）** | 推理步骤建模为图，支持合并/分支 | 需要关联多信息的推理 |
| **Self-Ask** | 显式提出子问题逐个解答 | 事实核查、复杂查询 |

### Agent 与 RAG 的关系

- **RAG**：解决"知识"问题，是模型的记忆扩展
- **Agent**：解决"能力"问题，是模型的行为扩展
- **协同**：高级系统中，Agent 可将 RAG 作为其核心工具之一来调用。RAG 也可引入 Agent 思维管理多源检索。
