## 1. 全景概括

### 逐话题概括

本对话是一份用户与「元宝」AI 助手之间、跨越 25+ 轮问答的完整技术对话记录，话题从宏观到微观、从概念到工程实践、从单点技术到系统架构逐步深入。以下按话题出现顺序逐一概括：

**话题 1（第 1 节）：大模型应用开发核心问题**——元宝给出顶层回答：大模型应用开发的核心是将通用 AI 能力转化为特定场景的实际价值，解决四个问题：场景适配（提示工程/微调/知识增强）、效率与成本（缓存/蒸馏/量化/混合系统）、可控性与可靠性（验证机制/内容过滤/流程管控）、集成与工程化（API 封装/状态管理/分布式调度）。

**话题 2（第 2 节）：技术角度五大问题**——从技术视角细分为五类：模型能力定制与增强（提示工程/微调/RAG）、性能成本与规模化（推理优化/缓存/成本控制）、可控性安全与合规（输出约束/内容过滤/幻觉缓解）、系统架构与工程化（服务化/状态管理/可观测性）、数据与评估（评估体系/数据闭环）。

**话题 3（第 3 节）：技术问题双重分类**——横向按技术模块分（模型层/应用层/服务架构层/安全合规层/数据评估层）和纵向按开发流程分（规划定义→原型验证→工程化部署→评估运维迭代）。两者交织使用：流程分类提供「何时做」的路线图，模块分类明确「做什么」和「谁来做」。

**话题 4（第 4 节）：RAG vs Agent 核心区别与联系**——RAG 解决「知识不足和不准」（静态知识局限/幻觉），Agent 解决「能力不足和自动化」（无法操作外部系统/处理多步骤任务）。RAG = 知识库扩展（检索→增强→生成流水线），Agent = 行为与流程扩展（感知→规划→执行循环）。两者互补：Agent 可调用 RAG 作为工具，RAG 可引入 Agent 思维管理复杂检索流程。

**话题 5（第 5-6 节）：Function Calling 与工具自主选择**——大模型本身不能直接自主选择工具，需通过 Agent 系统实现。核心流程：工具定义与暴露 → 任务理解与工具匹配（模型输出结构化决策如 JSON）→ 调度器解析并执行 → 结果整合与循环。支撑技术：Function Calling API（标准化接口/意图识别/参数提取）、提示工程（ReAct 模式）、Agent 框架（LangChain/LlamaIndex/AutoGen/CrewAI）。深入讨论了「仅 Function Calling」vs「完整 Agent 系统」的区别：前者是「按需调用」（单次被动），后者是「自主规划」（思考→行动→观察循环 + 记忆与状态管理 + 工具路由与组合）。

**话题 6（第 7-8 节）：意图识别在 Agent 中的角色**——额外的意图识别模块本质是将「是否用工具、用哪个工具」的决策权从大模型部分转移到一个更轻量、更可控的模块上，原因有四：成本与延迟优化（小模型粗筛→大模型精选）、稳定性与可控性（确定性过滤器）、处理简单高频请求的效率（直接路由绕过 LLM）、架构解耦与团队协作。两种模式对比：「纯大模型驱动」（灵活但昂贵）vs「意图识别+大模型」（高效但受限于预定义意图）。深入讨论「先意图识别还是先大模型」：取决于工具数量/任务开放性/性能要求/团队能力，现代最佳实践是混合动态路由（短路策略/分级路由/大模型兜底）。

**话题 7（第 9-12 节）：单智能体 vs 多智能体**——单智能体 = 全能专家（中心化决策），多智能体 = 专业团队（去中心化协作）。单智能体 = 大模型 + 系统提示词 + 工具调用 + 自主工作流（规划→执行→反思）。多智能体 = 多个单智能体结构 + 系统级协作机制（通信协议 + 任务协调 + 共享状态）。两种协调模式：管理者-工作者（中心化）和自主协作（去中心化）。个体区分靠角色定义/专属工具/记忆隔离，交互靠结构化消息/共享黑板/消息总线。

**话题 8（第 13 节）：双向量数据库架构**——单一向量检索在精度和召回率间难以权衡，双库架构用 Dense Embeddings（语义/意图匹配）和 Sparse Embeddings（关键词/实体精确匹配）分工互补。实现分两阶段：双路编码（同一文本块生成 Dense + Sparse 向量分存两个库）→ 双路并行检索 + 结果融合（RRF 或加权线性组合）→ 重排序返回。

**话题 9（第 14 节）：PDF 复杂版面处理**——详细讨论了跨页表格合并、分栏文本阅读顺序重建、复杂版面分析的工程实现，含完整代码示例。推荐工具栈：Camelot/Tabula/PDFPlumber（表格）、PaddleOCR/Tesseract（OCR）、版面分析模型。

**话题 10（第 15 节）：RAG 切块策略大全**——覆盖 6 种策略：固定大小切块（最简单但效果最差）、递归字符切块（LangChain 默认，最常用）、语义切块（基于相似度阈值，最优但计算成本高）、按标题/章节切块（利用文档层级结构）、表格特殊处理、图片处理。附带完整 Python 代码实现。

**话题 11（第 16 节）：混合检索**——向量检索 + 关键词检索 + 元数据过滤的协同工作。向量是「理解者」（语义含义），关键词是「匹配者」（精确词汇），元数据是「过滤者」（结构化条件）。三者融合策略：加权分数融合、RRF、元数据过滤交集、重排序。含完整代码示例（电商场景）。

**话题 12（第 17 节）：Agent 范式大全**——系统梳理 12 种范式：ReAct（推理+行动循环）、CoT（逐步推理）、ToT（多路径探索树）、GoT（图推理）、Plan-and-Execute（先规划后执行）、Reflexion（从错误学习）、CrewAI（角色分工）、AutoGen（对话协商）、CAMEL（角色扮演）、Code Agent（软件开发全流程）、Web Agent（网页操作）、Self-Ask/AoT（新兴混合范式）。附选择指南和代码示例。

**话题 13（第 18 节）：重排序技术**——初步检索（向量相似度）后加精排层，能提升 MRR 15-40%、幻觉率降低 20-30%。两种核心方法：交叉编码器（Cross-Encoder，联合编码查询+文档）和学习排序（LTR，多特征组合）。含 BGE-reranker 代码实现。

**话题 14（第 19 节）：知识图谱 RAG vs 稀疏检索**——稀疏矩阵检索（BM25）是底层检索算法（关键词匹配），知识图谱 RAG 是上层系统架构（关系推理+语义理解+查询扩展）。两者协同：图谱负责检索前的智能扩展和推理（查询增强器），BM25/稠密检索负责从海量文本快速初筛（召回器），最终形成「三驾马车」——稀疏检索 + 稠密检索 + 知识图谱推理。

**话题 15（第 20 节）：知识图谱与 Agent 结合**——超越 RAG 的结合点：作为 Agent 的规划与决策推理引擎（状态表示→路径规划→决策支持）、作为长期记忆与个性化核心存储（结构化记忆关联）、作为多智能体协作的共享认知层（共享世界模型/知识驱动的任务分解与协调）。

**话题 16（第 21 节）：Agent Skill 文档结构**——skill.json（元数据：名称/描述/参数/依赖/权限）+ main.py（纯技术实现，不包含业务规则）+ config.yaml（技术配置）+ test.py（功能正确性验证）。核心设计原则：Skill 文档只描述「如何调用/如何实现」（说明书），知识图谱定义「何时调用/为何调用」（施工规范），意图识别理解「用户想要什么」（业主需求）。三者职责严格分离。

**话题 17（第 22 节）：完整系统架构总结**——元宝对前 21 个话题进行全景回顾，产出系统架构全景图（意图识别→元认知决策→RAG 路径/Agent 路径→输出）。深度解析各组件：意图识别（翻译官与调度员）、元认知决策层、RAG 路径（向量数据库 + 重排序 + LLM 生成）、Agent 路径（知识图谱规则推理 + 工作流引擎 LangGraph + Agent Skills）。

**话题 18（第 23-24 节）：Harness Engineering**——核心思想从「优化输入」转向「管控系统」：LLM 是拥有蛮力的野马，Harness（马具）是缰绳/鞍具/导航系统。基础设施（而非模型智能）是当前 AI 应用落地的真正瓶颈——仅优化 Harness（工具格式/反馈循环）就能让同一模型表现提升数倍至数十倍。六大组件：文件系统（工作空间+版本控制）、Bash+沙箱（代码执行与自我验证）、记忆 AGENTS.md（跨会话长期记忆）、Web Search+MCP（实时知识+工具连接）、上下文工程（高信噪比输入管理）、编排+Hooks（多 Agent 协同+确定性检查）。

**话题 19（第 25 节）：Prompt/Context/Harness Engineering 三者关系**——层层嵌套、递进互补。Prompt Engineering（单次输入指令设计）⊂ Context Engineering（输入信息组织与供给）⊂ Harness Engineering（执行全链路系统管控）。比喻：PE=教司机怎么开车，CE=给司机地图和路况，HE=建造整条高速公路系统。

**话题 20（第 26-27 节）：四大智能体平台对比**——Hermes（自进化伙伴，Kanban Swarm 动态工作流，五层记忆体系，技能自动生成）、OpenClaw（全能网关，ChannelPlugin 架构，ClawHub 技能市场，模块化文件组装）、Claude Code（深度工程伙伴，并行动态工作流，MCP 生态 800+ 服务器，子 Agent 摘要压缩）、Codex（企业工作平台，预设角色驱动，深度集成 62 个企业应用，站点化成果）。全方位对比 7 个维度：工作流编排/记忆机制/文件系统/上下文工程/感知扩展/安全沙箱/典型场景。
