## 说明

本目录保存了 KnowledgeFlow SOP 体系的 v1.0 原始版本，仅供历史参考。v1.0 的核心局限性：

- **提取与策展合并在同一 SOP**——Agent 必须同时做穷举提取和重要性判断，这两个认知任务是冲突的
- **无人类审查面**——从原料直接跳到 wiki 页面，中间推理过程完全在 Agent 黑箱中
- **无结构化提取接口**——提取产物没有标准格式，无法逐条审计
- **标签注册死锁**——新原料需要新标签，但新标签需先在 SCHEMA 注册，形成鸡和蛋的矛盾

这些局限驱动了 v2.0 的架构重构：将提取和策展拆分为独立 SOP，中间插入阅读地图作为人类审查面，并引入 7 条硬约束确保 Agent 行为边界不被模糊化。

详细的设计决策和变更对照见 [CHANGELOG.md](../../CHANGELOG.md)。
